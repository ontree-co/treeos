Current Status

The PostgreSQL database for nzyme has been successfully deployed. However, the nzyme application itself requires manual installation due to:

  • The community Docker image (r4ulcl/nzyme) uses an outdated version (v1.2.2 from 2021)
  • Network monitoring requires special permissions that are restricted in containers
  • Modern nzyme versions (v2.0+) have significant improvements but lack official Docker support

Database Connection Details

PostgreSQL is ready with these credentials: 
Host: localhost
Port: 5433
Database: nzyme
Username: nzyme
Password: nzyme_password

Recommended Installation Options

Option 1: Direct Installation on Host (Recommended)

Install nzyme directly on your Ubuntu/Debian system following the official guide:

  1. Visit Official Installation Guide
  2. Download the latest .deb packages
  3. Configure to use the PostgreSQL database above
  4. Set up network interfaces for monitoring

Option 2: Build Custom Docker Image

Create a modern Docker image with nzyme v2.0+:

FROM ubuntu:24.04
RUN apt-get update && apt-get install -y openjdk-17-jdk wget
# Download and install latest nzyme
# Configure for container environment

Option 3: Run Taps Externally

Deploy nzyme-tap on dedicated hardware (Raspberry Pi, etc.) with proper network access for monitoring.

Network Monitoring Limitations

Important: Full network monitoring capabilities require:
  • NET_ADMIN capability (restricted in TreeOS for security)
  • Access to network interfaces in monitor mode
  • WiFi adapter with monitor mode support (for WiFi monitoring)

Next Steps

  1. Choose your installation method from the options above
  2. Configure nzyme to connect to the PostgreSQL database
  3. Set up network interfaces for monitoring (if running on host)
  4. Access the nzyme web interface (typically on port 22900)
